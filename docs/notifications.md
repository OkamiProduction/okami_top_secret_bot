# 📨 Уведомления в Telegram

В проекте настроена система уведомлений, которая информирует команду о ключевых событиях в процессе разработки: прохождении CI/CD, результатах ревью и слиянии Pull Request'ов. Все сообщения приходят в общий Telegram-чат, что делает процесс максимально прозрачным.

---

## 🤖 Используемые Telegram-боты

В проекте задействовано два разных бота:

| Бот | Назначение | Где используется |
|-----|------------|------------------|
| **Основной бот** (`OkamiTopSecretBot`) | Выполнение контрольных работ, взаимодействие с пользователями | Запускается на сервере |
| **Бот уведомлений** (например, `OkamiDeployNotifierBot`) | Отправка служебных сообщений о CI/CD, ревью, мёрджах | GitHub Actions workflows |

**Почему два бота?** Это разделяет зоны ответственности: рабочий бот может перезапускаться, обновляться или временно отключаться, в то время как уведомления должны приходить стабильно и независимо от состояния основного сервиса.

## 🛠️ Настройка бота уведомлений

### 1. Создание бота

1. Напишите [@BotFather](https://t.me/BotFather) в Telegram.
2. Отправьте команду `/newbot`.
3. Укажите имя (например, `Okami Deploy Notifier`) и username (например, `OkamiDeployNotifierBot`).
4. Скопируйте полученный **токен** (вида `1234567890:ABCdef...`).

### 2. Создание группы и добавление бота

1. Создайте новую группу в Telegram.
2. Добавьте в неё всех разработчиков, которые должны получать уведомления.
3. Добавьте в группу созданного бота (найдите его по username).
4. **Обязательно сделайте бота администратором группы**, иначе он не сможет отправлять сообщения.

### 3. Получение `chat_id`

Самый простой способ — добавить в группу бота [@RawDataBot](https://t.me/RawDataBot) и отправить любое сообщение. В ответ придёт JSON, в котором будет поле `chat.id` (обычно отрицательное число, например `-1001234567890`).

Скопируйте это значение — оно потребуется для секрета `TELEGRAM_NOTIFY_CHAT_ID`.

### 4. Добавление токена и chat_id в GitHub Secrets

В репозитории перейдите в **Settings → Secrets and variables → Actions** и добавьте:

| Секрет | Значение |
|--------|----------|
| `TELEGRAM_NOTIFY_BOT_TOKEN` | Токен бота уведомлений |
| `TELEGRAM_NOTIFY_CHAT_ID` | ID чата (группы) |

## 📬 Типы уведомлений и их источники

Все уведомления отправляются с помощью Action [appleboy/telegram-action](https://github.com/appleboy/telegram-action).

| Workflow | Событие | Эмодзи | Пример сообщения |
|----------|---------|--------|------------------|
| `ci.yml` | Успешное прохождение CI | ✅ | `✅ *CI Success*` |
| `ci.yml` | Провал CI | ❌ | `❌ *CI Failed*` |
| `deploy.yml` | Успешный деплой на staging | ✅ | `✅ *CI/CD Success*` |
| `deploy.yml` | Ошибка деплоя | ❌ | `❌ *CI/CD Failed*` |
| `notify-merge.yml` | Успешный мёрдж PR | 🎉 | `🎉 *Merged Successfully*` |
| `notify-review.yml` | Approve ревью | ✅ | `✅ *Approved*` |
| `notify-review.yml` | Request changes | 🔴 | `🔴 *Requested Changes*` |
| `notify-review.yml` | Комментарий | 💬 | `💬 *Commented*` |

## 📝 Примеры сообщений

### ✅ CI Success (при открытии/обновлении PR)

```text
✅ *CI Success*
[PR #42 — Добавлена команда /weather](https://github.com/OkamiProduction/okami_top_secret_bot/pull/42)
Author: okami
```

### ❌ CI/CD Failed (ошибка деплоя после мёрджа)

```text
❌ *CI/CD Failed*
Commit: `abc1234`
Author: okami
[View logs](https://github.com/OkamiProduction/okami_top_secret_bot/actions/runs/1234567890)
```

### 🎉 Merged Successfully

```text
🎉 *Merged Successfully*
[PR #42 — Добавлена команда /weather](https://github.com/OkamiProduction/okami_top_secret_bot/pull/42)
Author: okami
Merged by: okami
```

### ✅ Approved

```text
✅ *Approved*
[PR #42 — Добавлена команда /weather](https://github.com/OkamiProduction/okami_top_secret_bot/pull/42)
Reviewer: hermione
```

### 🔴 Requested Changes

```text
🔴 *Requested Changes*
[PR #42 — Добавлена команда /weather](https://github.com/OkamiProduction/okami_top_secret_bot/pull/42)
Reviewer: hermione
Нужно поправить обработку ошибок.
```

## 🎨 Кастомизация сообщений

Все шаблоны сообщений находятся в соответствующих `yml`-файлах в `.github/workflows/`. Вы можете легко изменить текст, добавить новые переменные или изменить эмодзи.

**Важно:** GitHub Actions автоматически маскирует секреты в логах, поэтому токены и ключи никогда не попадут в открытый вид, даже если случайно вывести их в сообщение (но в шаблонах они не используются напрямую, только через `secrets` контекст).

## 🔧 Устранение неполадок

### Уведомления не приходят

1. **Проверьте, что бот добавлен в группу и является администратором.**
2. **Проверьте правильность `TELEGRAM_NOTIFY_CHAT_ID`**. ID чата супергруппы начинается с `-100`.
3. **Посмотрите логи workflow в GitHub Actions**. Если Action `appleboy/telegram-action` завершился успешно, но сообщения нет — проблема на стороне Telegram (например, бот заблокирован).
4. **Убедитесь, что токен бота верен.** Можно временно добавить шаг в workflow для отправки тестового сообщения вручную.

### Дублирование уведомлений

Workflow `ci.yml` и `deploy.yml` настроены так, чтобы отправлять уведомления **только** при соответствующих триггерах. Если вы заметили дубли, проверьте, не срабатывают ли оба workflow на одно событие (например, при пуше в PR ветку). По умолчанию `ci.yml` срабатывает на `pull_request`, а `deploy.yml` — на `push` в `main`, поэтому конфликтов быть не должно.
