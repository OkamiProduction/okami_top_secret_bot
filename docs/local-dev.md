# 🖥️ Локальная разработка

Этот документ описывает всё, что нужно знать для работы с кодом бота на вашей локальной машине: от структуры проекта до добавления новых команд и отладки.

---

## 📁 Структура проекта

Проект следует принципам **Чистой архитектуры** (Clean Architecture). Бизнес-логика изолирована от внешних зависимостей (Telegram API, файловая система).

```text
okami_top_secret_bot/
├── main.go                     # Точка входа, сборка зависимостей
├── internal/
│   ├── config/
│   │   └── config.go           # Загрузка и валидация переменных из .env
│   ├── logger/
│   │   └── logger.go           # Инициализация slog с ротацией (lumberjack)
│   └── bot/
│       └── bot.go              # Логика бота, обработка сообщений
├── deploy/
│   └── tgbot.service           # Systemd unit-файл для сервера
├── scripts/
│   └── clean_logs.sh           # Очистка логов на сервере
├── .env.example                # Шаблон переменных окружения
├── .gitignore
├── go.mod / go.sum
└── docs/                       # Документация
```

### 🔍 Назначение ключевых пакетов

| Пакет | Ответственность | Зависимости |
|-------|-----------------|-------------|
| `config` | Чтение `.env`, установка значений по умолчанию. | `godotenv` |
| `logger` | Создание настроенного `*slog.Logger` с двумя выходами (консоль + файл) и ротацией. | `slog`, `lumberjack` |
| `bot` | Взаимодействие с Telegram API, обработка команд, отправка ответов. **Не зависит** от `config` и `logger` напрямую (они передаются через конструктор). | `go-telegram-bot-api/v5` |
| `main` | Собирает все компоненты вместе: загружает конфиг, создаёт логгер, запускает бота с graceful shutdown. | Все внутренние пакеты |

## ⚙️ Переменные окружения (`.env`)

Файл `.env` **не коммитится** в репозиторий. Используйте `.env.example` как основу.

| Переменная | Обязательная | По умолчанию | Описание |
|------------|--------------|--------------|----------|
| `TELEGRAM_TOKEN` | ✅ Да | — | Токен бота, полученный от @BotFather |
| `DEBUG` | ❌ Нет | `false` | Включает подробный лог запросов к Telegram API |
| `LOG_LEVEL` | ❌ Нет | `info` | Уровень логирования: `debug`, `info`, `warn`, `error` |
| `LOG_FILE` | ❌ Нет | `./bot.log` | Путь к файлу для JSON-логов (на сервере: `/var/log/tgbot/bot.log`) |

## 🚀 Запуск и остановка

### Первый запуск

1. Убедитесь, что Go установлен (`go version` → 1.22+).
2. Скопируйте `.env.example` → `.env` и пропишите реальный токен.
3. Выполните в корне проекта:

   ```bash
   go mod tidy
   go run main.go
   ```

После запуска в консоли появится сообщение `Бот авторизован username=...`. Бот начнёт принимать сообщения через **Long Polling**.

### Остановка

Нажмите `Ctrl+C` в терминале. Благодаря graceful shutdown (контекст с `signal.NotifyContext`) бот дождётся завершения текущих операций перед выходом.

## ✏️ Добавление новых команд

Вся логика обработки сообщений сосредоточена в методе `handleCommand` файла `internal/bot/bot.go`.

### Пример: добавляем команду `/ping`

1. Откройте `internal/bot/bot.go`.
2. Найдите метод:

   ```go
   func (b *Bot) handleCommand(text string) string {
       switch text {
       case "/start":
           return "👋 Привет! Я бот на Go."
       case "/help":
           return "ℹ️ Пока умею только /start и /help."
       default:
           return "Неизвестная команда. Попробуйте /help"
       }
   }
   ```

3. Добавьте новый `case`:

   ```go
   case "/ping":
       return "🏓 Pong! " + time.Now().Format(time.RFC3339)
   ```

4. Перезапустите бота и проверьте командой `/ping`.

### Более сложные обработчики

Если логика команды требует нескольких шагов (например, запрос к внешнему API, работа с базой данных), рекомендуется:

- Выделить отдельный метод в структуре `Bot` (например, `handleWeatherCommand`).
- Использовать `b.logger.Info` / `b.logger.Error` для логирования.
- Для фоновых задач — запускать горутины, но **обязательно** контролировать их завершение через контекст (передаётся в `Start`).

## 🧪 Локальная отладка

### Логи

- **Консоль:** все сообщения выводятся в формате обычного текста.
- **Файл:** дублируются в JSON-формате по пути из `LOG_FILE`. Для просмотра удобно использовать `tail -f bot.log`.

### Включение отладки Telegram API

Установите в `.env`:

```env
DEBUG=true
```

После перезапуска в логах появятся полные HTTP-запросы и ответы к `api.telegram.org`.

### Работа с точками останова

Проект легко отлаживать в любой IDE (GoLand, VS Code с Go-плагином). Просто запустите `main.go` в режиме отладки.

## 🧱 Принципы для дальнейшего расширения

При добавлении новых возможностей старайтесь придерживаться следующих правил, заложенных в архитектуру:

1. **Бизнес-логика — в `internal/bot` (или отдельных use-case пакетах).** Не пишите сложную логику в `main.go`.
2. **Зависимости передавайте явно.** Если боту нужен HTTP-клиент или репозиторий, передавайте их через конструктор `New()`.
3. **Логируйте структурированно.** Используйте `slog.Info` с ключами `"user", "chat_id", "command"`.
4. **Не забывайте про graceful shutdown.** Если запускаете горутины, используйте переданный `context.Context` для их завершения.
